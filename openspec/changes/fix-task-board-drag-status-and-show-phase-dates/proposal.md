## Why

当前任务看板中生成的任务块无法通过拖拽跨列变更任务状态，导致用户必须通过其他入口手动修改状态，降低看板操作效率。同时任务块缺少阶段与起止时间信息，用户在看板视图中无法快速判断任务所处阶段和排期。

## What Changes

- 为任务看板任务块增加拖拽交互，支持通过拖拽到不同状态列来变更任务状态。
- 为任务看板任务块补充阶段信息展示，便于快速识别任务所属阶段。
- 为任务看板任务块补充起止时间展示，提升排期可见性与看板决策效率。
- 明确拖拽状态变更的前后端交互行为与失败反馈要求，确保状态更新一致性。

## Capabilities

### New Capabilities
- `task-board-drag-status-transition`: 支持在任务看板中通过拖拽任务块到目标列来触发任务状态变更并完成持久化更新。
- `task-card-phase-and-date-display`: 在任务看板任务块中展示任务阶段、开始时间和结束时间（含缺失值时的显示规则）。

### Modified Capabilities
- （空）

## Impact

- Frontend: `frontend/src/features/` 下任务看板相关页面与任务卡片组件，需要增加拖拽交互和字段展示。
- Frontend API client: 可能需要复用或补充任务状态更新接口调用与错误处理逻辑。
- Backend API: 若现有接口未满足看板拖拽更新需求，需补充任务状态更新校验或返回字段。
- 数据模型/返回字段: 需要确保任务列表接口返回阶段、开始时间、结束时间等字段供看板卡片展示。

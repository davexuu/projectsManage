## 变更说明与使用指引

### 1) 新增后端接口：规则拆解建议

- Endpoint: `POST /api/wbs/quick-suggestions`
- 入参：
  - `projectId: string`
  - `prompt: string`（“我要做什么”，2-200 字）
- 出参：
  - `intent`: `新增 | 修复 | 优化 | 合规`
  - `reason`: 命中规则说明
  - `items`: 可直接进入批量录入列表的建议分解行

规则未命中时默认回退 `新增` 模板，保证流程可继续人工编辑。

### 2) 自动编码策略

- 单条创建与批量创建均支持自动生成 `wbsCode`
- 当请求未传 `wbsCode` 时，系统按 `阶段序号.工作包序号.任务序号` 生成
- 同步为缺失 `sortOrder` 的记录生成排序值，保证查询排序稳定
- 兼容已有手工编码：若传入合法 `wbsCode`，系统保留原值

### 3) 快速分解前端体验

在计划编排工作台 `快速分解WBS` 弹窗中：

- 新增“快速生成建议”卡片：输入一句话后点击“生成建议”
- 建议结果直接加入“分解项列表”，可继续编辑/删除
- `WBS编码` 改为自动只读展示
- 关键字段说明改为信息图标悬浮查看
- 前置任务改为名称多选（非手填 ID）

### 4) 依赖关系与冲突提示

- 提交前先调用 `validate-plan`，冲突按“行号 + 字段 + 原因”回显
- 冲突行在弹窗列表中高亮
- 规划工作台 WBS 列表增加“紧前任务/紧后任务”列，支持双向可见

### 5) 回归项

已补充 API 回归脚本断言：
- 规则拆解建议接口可用
- WBS 单条自动编码
- WBS 批量自动编码

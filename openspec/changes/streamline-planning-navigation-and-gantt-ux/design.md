## Context

当前规划链路存在两套并行入口：
- 项目空间直接入口：`/planning-studio`
- 规划过程子入口：`/module/wbs`、`/module/milestones`、`/gantt`

在系统尚未上线且允许直接演进信息架构的阶段，继续保留兼容入口会放大导航歧义。与此同时，计划编排工作台内部存在可用性不一致：阶段筛选为下拉选择、时间范围占位未中文化、列表日期格式在多处不统一；甘特依赖线缺少显式语义提示，用户容易将其误读为随机连线或异常数据。

约束：
- 不新增后端接口，优先通过前端路由、组件与展示层改造完成。
- 规划功能主链路必须可在单入口完成（筛选-列表-图表联动不破坏）。
- 变更期间允许直接移除兼容入口，不保留双轨。

## Goals / Non-Goals

**Goals:**
- 将规划入口收敛到“规划过程 -> 计划编排工作台”单一路径。
- 移除兼容入口菜单，避免用户误入旧页面。
- 将阶段筛选升级为按钮式点选，提高切换效率。
- 统一时间范围占位文案为中文，提升一致性。
- 统一工作台表格内日期显示为 `YYYY-MM-DD`。
- 在甘特图中补充依赖线语义说明，让用户理解“前置-后继”关系。

**Non-Goals:**
- 不改造后端数据模型与依赖计算逻辑。
- 不重构甘特图库（继续使用 `frappe-gantt`）。
- 不引入自动排程、资源平衡等高级排程能力。

## Decisions

### 1) 导航收敛到规划过程
决策：在“规划过程”下新增/替换为“计划编排工作台”入口，原 `WBS/里程碑/甘特` 从规划主导航移除；同时移除系统“兼容入口”菜单组。

原因：
- 单入口减少心智分叉，符合“规划在同一工作台完成”的产品语义。
- 系统未上线，迁移成本低于后续补救成本。

备选：
- 保留旧入口并添加提示跳转。短期平滑但会长期保留歧义。
- 直接收敛，配合一次性回归验证，路径最清晰。

### 2) 阶段筛选改为按钮组
决策：将阶段筛选从 `Select` 改为按钮组（含“全部”），单击即切换。

原因：
- 阶段集合固定且数量少（启动/规划/执行/验收），按钮交互效率高。
- 降低下拉展开与滚动成本，适合高频切换场景。

备选：
- 保持下拉，增加快捷键。复杂度更高且可见性较差。

### 3) 日期与文案规范统一
决策：
- 时间范围控件占位统一中文（如“开始日期”“结束日期”）。
- 规划工作台表格日期统一格式化为 `YYYY-MM-DD`（不展示时分秒）。

原因：
- 统一展示规则有助于比对排期与减少视觉噪声。
- 与甘特主时间粒度一致，避免跨视图认知跳变。

备选：
- 保留原始日期字符串。实现最省但用户感知最差。

### 4) 甘特依赖线语义显式化
决策：保留依赖线渲染机制，增加“依赖线含义”可视说明（图例/提示文案/悬浮说明之一或组合），明确其表示“前置任务 -> 当前任务”。

原因：
- 当前依赖线存在但语义不显式，导致误解。
- 不改库可快速补足可解释性。

备选：
- 自绘依赖线交互层。可控性更高但投入过大，不符合本次范围。

## Risks / Trade-offs

- [风险] 直接移除兼容入口后，已有书签路径访问体验变化 → Mitigation：路由层增加必要重定向或空态引导。
- [风险] 按钮筛选引入状态同步问题（与甘特/列表联动）→ Mitigation：保持单一 `stage` 状态源并补联动回归用例。
- [风险] 日期统一格式可能掩盖时区/时分信息 → Mitigation：规划页仅展示日粒度，明细弹层保留原值能力。
- [风险] 依赖线说明过重影响界面简洁度 → Mitigation：采用轻量提示（图例+hover 文案），避免大段常驻说明。

## Migration Plan

1. 导航调整：重组规划菜单项，加入“计划编排工作台”，移除兼容入口菜单。
2. 交互调整：在工作台替换阶段筛选控件并统一时间占位中文。
3. 展示统一：统一 WBS/里程碑表格日期渲染格式为 `YYYY-MM-DD`。
4. 甘特可解释性：补依赖线说明并校验与现有依赖映射一致。
5. 回归验证：执行“入口可达、筛选联动、日期格式、依赖线提示”四类前端回归。

回滚策略：
- 若出现导航阻断，临时恢复规划子入口配置；
- 若交互改造导致联动异常，回退至原筛选组件与默认占位，同时保留已完成 spec 文档。

## Open Questions

- 依赖线语义说明采用何种最轻方式（图例文本、hover tooltip、帮助 icon）更符合当前视觉风格？
- 旧路径是否全部重定向到工作台，还是对少量内部调试路径保留直达？

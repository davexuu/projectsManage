## Why

当前 WBS 任务分解录入流程存在交互逻辑不清晰、字段负担偏重、填写要求不明确的问题，导致开发者在为项目维护 WBS 人物分解项时效率低、出错率高。同时现有表格样式可读性与一致性较差，影响日常管理体验。

## What Changes

- 重构 WBS 任务分解录入交互，提供更符合开发者填写习惯的流程（按项目上下文、分步填写、减少重复输入）。
- 优化“新增记录”表单字段，移除或降级非必要字段（改为可选/自动推导），降低录入成本。
- 为关键字段增加填写要求/格式说明（如必填、示例、取值约束、校验提示），提升填写准确性。
- 统一全站表格样式与交互表现，升级为接近 ElementUI Table 的视觉与体验（表头、斑马纹、悬浮态、边框、间距、空状态等）。
- 明确 WBS 列表与录入页面的样式规范与字段呈现规范，保证后续页面一致性。

## Capabilities

### New Capabilities
- `wbs-entry-flow`: 提供面向项目的 WBS 分解项便捷录入流程与交互规范（步骤、默认值、联动、减少重复输入）。
- `wbs-form-guidance`: 为 WBS 新增/编辑表单提供字段级填写要求、示例文案与校验提示规则。
- `table-ui-standard`: 提供统一的业务表格样式标准与可复用表格展示行为（接近 ElementUI Table 的体验）。

### Modified Capabilities
- None currently (no existing spec capabilities under `openspec/specs/` to modify).

## Impact

- 影响 WBS 任务分解相关页面（列表、创建/编辑弹窗或表单）。
- 影响通用表格样式层（全局样式、组件样式或表格封装组件）。
- 可能影响表单校验规则、字段定义配置、前端交互逻辑及部分接口字段映射。
- 需要产品/业务确认哪些“新增记录字段”为非必要字段，以及填写说明文案标准。

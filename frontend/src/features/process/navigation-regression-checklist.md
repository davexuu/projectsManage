# 过程导航回归用例

## 1. 路由激活态

- 访问 `/process/start`，确认“启动”菜单高亮，过程页内导航激活项与当前路径一致。
- 直接打开深链 `/process/monitor/reports`，确认左侧“监控”高亮，过程页内“周报月报”激活。
- 在过程内跳转后使用浏览器后退/前进，确认激活项随路由变化。

## 2. 桌面端导航

- 在宽屏下展示分组导航，不同分组项可点击切换。
- 当前项需有高亮，最近访问区可快速回到最近路径。
- “最近访问”最多展示 5 项，顺序按最近访问时间。

## 3. 移动端导航

- 窄屏下显示“打开过程导航”入口，点击后抽屉打开。
- 抽屉内能访问与桌面一致的全部入口。
- 抽屉选择后自动关闭并跳转对应页面。

## 4. 指标采集

- 触发菜单点击后在 `sessionStorage[pmp_process_nav_metrics]` 看到 clickSteps 递增。
- 路由到达目标后生成 records 记录（含 reachMs、clickSteps、flowKey）。
- 可在控制台调用 `getNavigationMetricsReport()` 与 `evaluateNavigationAcceptance()` 查看对比结果。

## 5. 跨过程切换

- 从启动切换到规划、执行、监控、收尾，检查所有入口均可达。
- 在任一过程切到兼容入口(`/module/*`)后返回过程页，激活态恢复正确。

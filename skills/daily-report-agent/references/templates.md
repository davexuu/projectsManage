# 每日日报模板与字段清单

按用户场景选择模板；未指定时优先使用“通用研发日报”。

## 输入字段清单（优先级从高到低）

- 日期/周期（例如 `2026-02-23`）
- 今日完成（动作 + 结果）
- 进行中事项（状态）
- 问题/风险（影响、依赖、是否需要协助）
- 明日计划（动作 + 目标）
- 关键数据（数量、进度百分比、版本号、环境）
- 协作信息（对接人、团队、外部依赖）

## 模板 1：通用研发日报（默认）

```md
## 今日完成
- [事项]：完成了[动作]，产出/结果为[结果]。
- [事项]：定位并处理[问题]，当前状态[状态]。

## 进行中与问题
- [事项]：当前进度[进度]，阻塞点[阻塞点]。
- 风险/依赖：[内容]；影响[影响]；需要支持[需要支持/无]。

## 明日计划
- [事项]：计划执行[动作]，目标[目标]。
- [事项]：与[对象]完成[联调/确认/评审]。

## 待确认项（可选）
- [信息缺口]
```

## 模板 2：领导汇报版（压缩）

```md
## 今日关键进展
- [高价值结果 1]（影响：[业务/项目影响]）
- [高价值结果 2]（影响：[效率/质量/风险变化]）

## 风险与需要协调
- [风险]：影响[影响]，建议[动作]，需要[协助方/资源]。

## 明日重点
- [重点动作 1]
- [重点动作 2]
```

## 模板 3：项目推进日报（里程碑导向）

```md
## 项目状态
- 今日状态：`正常 / 有风险 / 阻塞`
- 当前里程碑：[名称]（计划日期：[日期]）

## 今日推进
- [模块/任务]：完成[内容]，产出[文档/代码/接口/测试结果]。
- [模块/任务]：进度从[起点]推进至[终点]。

## 风险与依赖
- [风险]：影响里程碑[名称/日期]。
- [依赖]：等待[团队/人员]在[时间]前提供[内容]。

## 下一步计划
- [动作]（负责人：[姓名/角色]，目标时间：[时间]）
```

## 生成规则（落地）

- 优先将“做了什么”改写为“做了什么 + 结果”。
- 将“继续跟进”补充对象和目标，例如“继续跟进登录接口联调，目标完成鉴权异常定位”。
- 同一事项跨多段出现时合并，避免重复。
- 缺少事实时保留原文并标记 `待确认`，不要自行补数据。

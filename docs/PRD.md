## 1. 背景

当前项目管理以 Excel 为核心载体，虽能满足填报，但存在以下问题：
- 跨表关联弱：立项、WBS、里程碑、推进、状态、风险、变更之间一致性依赖人工维护。
- 过程不可控：字段填写要求、流程状态口径主要在模板说明和批注里，系统无法自动校验。
- 权限治理不足：缺少“角色 + 项目成员”双重权限模型，易出现越权查看/编辑。
- 决策视角不足：缺少项目级综合看板，管理层难以及时发现逾期与风险热点。

因此需要建设一套与 Excel 模板强一致的项目管理系统，实现“可录入、可校验、可追踪、可分析、可授权”。

## 2. 目标

### 2.1 业务目标

- 建立项目全生命周期主链路：
  - 项目立项卡
  - WBS 任务分解
  - 里程碑计划
  - 推进记录
  - 项目状态评估
  - 风险问题台账
  - 变更申请
- 保持对既有 Excel 模板的兼容导入，降低历史数据迁移成本。
- 明确权限边界，保障数据安全与职责可追溯。

### 2.2 产品目标

- 提供标准化表单录入与列表查询能力。
- 提供导入预览与一键入库能力，支持清洗和去重。
- 提供项目总览看板，输出项目关键健康指标。
- 提供系统底座能力（组织、字典、sys_* 实体）支持后续扩展。

### 2.3 实施目标（当前版本）

- 基于现有前后端架构稳定交付可用版本。
- 将 Excel 批注中的关键规则沉淀为可开发规则清单（见规则文档）。

## 3. KPI

### 3.1 业务 KPI

- 导入可用率：清洗后记录成功入库率 >= 95%。
- 去重有效率：WBS/里程碑/风险/变更重复数据拦截率 = 100%。
- 权限拦截率：越权访问与越权写入拦截率 = 100%。

### 3.2 产品 KPI

- 登录后 3 秒内进入主界面（本机标准联调环境）。
- 单条记录手工新增步骤 <= 6 步。
- 项目看板可稳定返回核心 KPI（任务、里程碑、风险、趋势）。

### 3.3 管理 KPI

- 每个项目可按周期输出状态评估并追溯变化趋势。
- 风险热点可按阶段汇总，用于例会管理。

## 4. 用户与用户故事

### 4.1 用户角色

- `ADMIN`：系统管理员
- `PM`：项目经理
- `MEMBER`：项目成员

### 4.2 用户故事

- 作为 `ADMIN`，我希望能维护系统用户、组织、字典和系统实体，以保障系统治理与运行。
- 作为 `ADMIN/PM`，我希望能够创建项目并维护全量项目过程数据，形成完整项目台账。
- 作为 `PM`，我希望通过 Excel 快速导入历史数据，减少重复录入成本。
- 作为 `MEMBER`，我希望在授权项目内提交推进记录并查看项目动态。
- 作为管理者，我希望在项目总览中一眼看到进度、逾期与风险状态，快速判断是否需要干预。

## 6. 关键业务规则

### 6.1 权限规则

- 所有业务接口需登录访问。
- 非 `ADMIN` 用户仅可读取其有项目权限的数据。
- 写入权限为“角色 + 项目成员权限”双重校验：
  - `WBS/里程碑/状态评估/风险/变更`：`ADMIN/PM` 且 `OWNER/EDITOR`
  - `推进记录`：`ADMIN/PM/MEMBER` 且 `OWNER/EDITOR`

### 6.2 数据校验规则（当前已落地主规则）

- 阶段：启动/规划/执行/验收
- 状态：未开始/进行中/已完成/延期
- 信号灯：绿/黄/红
- 是否：是/否
- 年度范围：2000~2100
- 进度百分比：0~100

### 6.3 导入规则（当前实现）

- 模板说明/示例行过滤。
- 日期序列值转标准日期格式。
- 项目按“项目名称 + 年度”复用或新建。
- 去重：
  - WBS：`taskName + level2WorkPackage`
  - 里程碑：`milestoneCode`
  - 风险：`riskCode`
  - 变更：`changeCode`
- 说明：更多 Excel 批注规则见规则文档，当前未全部程序化。

## 7. 非功能需求

- 前端：React + Vite + TypeScript
- 后端：Express + TypeScript + Zod
- 数据库：MySQL + Prisma
- 可观测性：至少保留健康检查接口 `/health`
- 错误语义：401（未登录）、403（权限不足）、400（参数错误）

## 8. 验收标准

### 8.1 功能验收

- ADMIN 能完成项目创建、过程模块录入、导入、看板查看、成员授权。
- PM 在授权项目内具备与职责匹配的录入能力。
- MEMBER 在授权项目内仅可提交推进记录（并满足成员权限要求）。
- 导入可返回预览摘要及入库创建/跳过统计。

### 8.2 权限验收

- 未登录请求受保护接口返回 401。
- 角色不匹配返回 403。
- 非授权项目读取或写入返回 403。

### 8.3 数据规则验收

- 所有枚举字段输入不合法值时，返回校验失败。
- 项目名称重复时阻断创建/更新。
- 看板指标与基础数据统计逻辑一致。

### 8.4 规则一致性验收

- Excel 批注规则中“本期选定规则集”实现率达到评审约定阈值（建议 >= 90%）。
- 未实现规则需在规则文档中标注状态与原因，不得隐性忽略。

## 9. 风险与依赖

- 风险：Excel 批注规则较多且存在跨表依赖，不做规则清单会导致实现偏差。
- 风险：部分字段在模板中存在多种写法（如错别字/别名），需统一口径。
- 依赖：业务方确认“强制规则”优先级与冲突处理策略。
